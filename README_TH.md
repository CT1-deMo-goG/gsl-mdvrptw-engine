# GSL Engine: โมดูล MDVRPTW ⏱️
เอนจินประมวลผลและเพิ่มประสิทธิภาพการจัดเส้นทางขั้นสูงที่มีความแน่นอนสูง (Deterministic) ออกแบบมาเพื่อแก้ปัญหาการจัดเส้นทางยานพาหนะแบบหลายคลังสินค้าและมีกรอบเวลา (Multi-Depot Vehicle Routing Problem with Time Windows - MDVRPTW)

**โมดูลแกนหลัก:** `GSL_MDVRPTW_V38` (สำหรับงานวิชาการ) & `GSL_MDVRPTW_REAL_V29` (สำหรับหน้างานจริง)

Repository นี้คือตัวแทนของสถาปัตยกรรมการจัดเส้นทางที่ซับซ้อนที่สุดในระบบ GSL Framework โดยสามารถจัดการกับคลังสินค้าหลายจุด, ข้อจำกัดด้านความจุแบบ 2 มิติ (น้ำหนัก/ปริมาตร), ระยะทางที่ไม่สมมาตร (ถนนวันเวย์), และกรอบเวลาการจัดส่งที่เข้มงวดไปพร้อมๆ กัน ซึ่งทั้งหมดนี้ประมวลผลเสร็จสิ้นบนสถาปัตยกรรมชิปเซ็ตมือถือ (Mobile Architecture)

⚠️ **ข้อสงวนสิทธิ์:** ซอร์สโค้ดของโปรเจกต์นี้เป็นความลับทางธุรกิจและไม่เปิดเผยแพร่สาธารณะ โปรเจกต์นี้เปิดรับเฉพาะความร่วมมือด้านงานวิจัยและการนำไปประยุกต์ใช้ในอุตสาหกรรมเท่านั้น

## 🏆 ไฮไลต์ผลงาน: การปรับแต่งโลจิสติกส์ในหน้างานจริง (ชุดข้อมูล Bosnia)
เพื่อพิสูจน์ศักยภาพในเชิงพาณิชย์ เอนจินนี้ได้ถูกนำไปทดสอบกับ **ชุดข้อมูลโลจิสติกส์จริงจากบริษัทจัดจำหน่ายขนาดใหญ่ในประเทศบอสเนียและเฮอร์เซโกวีนา** (ดึงข้อมูลพิกัดจริงผ่าน GraphHopper API/OpenStreetMap)

* **มาตรฐานเดิมของระบบบริษัท (Legacy System Baseline):** 537.43 กิโลเมตร
* **ระยะทางที่ปรับแต่งโดย GSL Engine V29:** 368.35 กิโลเมตร
* **ผลลัพธ์ทางธุรกิจ:** **ประหยัดระยะทางได้ 169.08 กิโลเมตรต่อรอบการส่ง (ลดต้นทุน 31.4%)**
* **เวลาที่ใช้ประมวลผล:** **0.0078 วินาที** (ใช้รถ 7 คัน)
* **ความถูกต้องของเงื่อนไข:** ผ่านเกณฑ์ 100% (ไม่มีการวิ่งย้อนศรวันเวย์, น้ำหนัก/ปริมาตรไม่เกิน, และไปถึงจุดหมายตรงตามกรอบเวลาเป๊ะ)

## 📊 พอร์ตโฟลิโอการทดสอบมาตรฐานวิชาการ (Academic Benchmark)
นอกเหนือจากข้อมูลโลกจริง เอนจินนี้ยังผ่านการทดสอบอย่างหนักหน่วงกับชุดข้อมูลทางวิชาการขนาดใหญ่:
1. **ชุดข้อมูลมาตรฐาน Cordeau (pr01-pr20):** ประมวลผลเสร็จสิ้นในระดับเสี้ยววินาที พร้อมความถูกต้องของเงื่อนไข 100%
2. **ชุดข้อมูลขนาดใหญ่ Vidal (สูงสุด 960 โหนด):** สร้างมาตรฐานใหม่ (Baseline) สำหรับการประมวลผลชุดข้อมูลขนาดใหญ่โตมโหฬารบนโทรศัพท์มือถือ

## บริการและความร่วมมือทางธุรกิจ

เปิดรับการร่วมงานและให้คำปรึกษาระดับมืออาชีพ ในขอบเขตดังต่อไปนี้:

- **Logistics-as-a-Service (LaaS):** บริการระบบประมวลผลเส้นทางแบบเรียลไทม์สำหรับฟลีทรถขนส่งระดับองค์กร
- **High-Precision Modeling:** การออกแบบและพัฒนาอัลกอริทึมเฉพาะทาง สำหรับแก้ปัญหาซัพพลายเชนที่มีเงื่อนไข (Constraints) ซับซ้อนขั้นสูง
- **Technical Consultancy:** บริการให้คำปรึกษาทางเทคนิค, ทดสอบขีดจำกัดระบบเครือข่ายขนาดใหญ่ (Stress-testing) และการตรวจสอบประเมินประสิทธิภาพการจัดเส้นทาง (Optimization audits)
